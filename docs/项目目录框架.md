### 目录结构
```
Top_Down_2D/
├── .git/
├── .gitignore             <-- 包含 Unity, Go, IDE 的忽略规则
├── README.md              <-- 如何启动服务器，如何运行客户端etc
├── docs/                  <-- 策划案、美术需求、技术架构图
├── proto/                 <-- 存放 .proto 文件 (协议定义)
│   ├── game.proto
│   └── gen_proto.bat/.sh  <-- 一键生成 Go 和 C# 代码的脚本
├── server/                <-- Go 服务端项目根目录
│   ├── go.mod
│   ├── go.sum
│   ├── cmd/               <-- 入口文件 (main.go)
│   ├── internal/          <-- 业务逻辑
│   │   ├── kcp_net/       <-- 网络层封装
│   │   └── game_logic/    <-- 核心战斗计算
│   └── pkg/               <-- 生成的 pb.go 文件放这里
└── client/                <-- Unity 项目根目录 
    ├── Assets/
    │   ├── Scripts/
    │   │   ├── Network/   <-- KCP 和 生成的 .cs 协议文件
    │   │   └── Gameplay/
    │   └── ...
    ├── Packages/
    └── ProjectSettings/
```


### Folder Structure
```
Top_Down_2D/
├── .git/
├── .gitignore             <-- Core! Must include ignore rules for Unity, Go, and IDEs simultaneously
├── README.md              <-- Clearly document: How to start the server and run the client
├── docs/                  <-- Game design docs, art requirements, technical architecture diagrams
├── proto/                 <-- [Core Contract] Store .proto files (protocol definitions)
│   ├── game.proto
│   └── gen_proto.bat/.sh  <-- One-click script to generate Go and C# code
├── server/                <-- Root directory of the Go server project
│   ├── go.mod
│   ├── go.sum
│   ├── cmd/               <-- Entry files (main.go)
│   ├── internal/          <-- Business logic
│   │   ├── kcp_net/       <-- Network layer encapsulation
│   │   └── game_logic/    <-- Core combat calculation
│   └── pkg/               <-- Store generated pb.go files here
└── client/                <-- Root directory of the Unity project (Note: not inside Assets)
    ├── Assets/
    │   ├── Scripts/
    │   │   ├── Network/   <-- KCP and generated .cs protocol files
    │   │   └── Gameplay/
    │   └── ...
    ├── Packages/
    └── ProjectSettings/
```
